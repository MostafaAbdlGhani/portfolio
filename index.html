The Codex — Game Design

Game Concept & Documentation

Graveborne — Design Overview

Core Loop: Explore → Combat → Die → Respawn at Grave → Grow stronger. Death is the mechanic. Graves serve as bonfires: respawn, fast travel, level up, unlock memory fragments.


Design Pillars: Weight (every hit must feel earned), Mystery (lore delivered through environment and item descriptions), Persistence (character build evolves across sessions via the Grave system).


Target audience: Fans of Dark Souls, Hollow Knight and Dead Cells who appreciate tight combat, interconnected worlds, and stories told without cutscenes.

Boss Design — Phase Documentation

The Fire Knight — Two-Phase Design

Phase 1 — Recognition: He fights with familiar swordsmanship. The player learns the moveset. The music is mournful, not aggressive. He is not a monster yet.


Phase 2 — Transformation: At 50% HP, fire consumes him. Speed increases, new attacks emerge, the music shifts. The player must unlearn what they just learned.


Design intent: Both Ashar and the Fire Knight were knights who were betrayed — but one succumbed to the fire. The fight is a mirror, not just an obstacle.

WeaponInstance.csC# / Unity
// Computes total Attack Rating (AR) from all damage profiles,
// then applies requirement + durability multipliers.
public float GetTotalAR(PlayerStats stats)
{
    if (data == null || data.damageProfiles == null) return stats.Strength;

    float totalAR = 0f;
    float reqMult = MeetsRequirements(stats) ? 1f : 0.4f;
    float durMult = GetDurabilityMultiplier();

    foreach (var profile in data.damageProfiles)
    {
        totalAR += GetDamageForType(profile, stats);
    }

    return totalAR * reqMult * durMult;
}

public Dictionary<DamageType, float> GetDamageBreakdown(PlayerStats stats)
{
    var breakdown = new Dictionary<DamageType, float>();

    if (data == null || data.damageProfiles == null)
    {
        breakdown[DamageType.Physical] = stats.Strength;
        return breakdown;
    }

    float reqMult = MeetsRequirements(stats) ? 1f : 0.4f;
    float durMult = GetDurabilityMultiplier();

    foreach (var profile in data.damageProfiles)
    {
        float damage = GetDamageForType(profile, stats) * reqMult * durMult;

        if (!breakdown.ContainsKey(profile.damageType))
            breakdown[profile.damageType] = 0f;

        breakdown[profile.damageType] += damage;
    }

    return breakdown;
}

private float GetDamageForType(WeaponDamageProfile profile, PlayerStats stats)
{
    float damage = profile.baseDamage;

    if (profile.scaling != null)
    {
        foreach (var s in profile.scaling)
        {
            if (s.rank == ScalingRank.None) continue;

            float statValue = stats.GetStatValue(s.stat);
            float scalingMult = GetScalingMultiplier(s.rank);
            damage += statValue * scalingMult;
        }
    }

    return damage;
}
BaseEnemy.csC# / Unity
// Enemy core loop: early-outs (dead/no player), then resolves retreat/stun,
// then runs a simple state machine (Patrolling → Chasing → Attacking/Blocking).
protected virtual void Update()
{
    if (state == EnemyState.Dead || player == null)
        return;

    slope?.CheckSlope();
    UpdatePhysicsMaterial();

    if (isRetreating) return;

    if (ShouldRetreat())
    {
        StartRetreat();
        return;
    }

    bool isPlayerInChaseBox = IsPlayerInBox(transform.position, enemyData.chaseBoxSize);
    bool isPlayerInAttackRange = IsPlayerInAttackRange();
    bool isAnyAttackReady = IsAnyAttackReady();

    if (state == EnemyState.Stunned)
    {
        if (Time.time >= stunEndTime)
            state = EnemyState.Chasing;
        return;
    }

    bool hasLOS = HasLineOfSightToPlayer();

    switch (state)
    {
        case EnemyState.Patrolling:
            if (isPlayerInChaseBox && !PlayerStats.Instance.PlayerIsDead && hasLOS)
                state = EnemyState.Chasing;
            else
                Patrol();
            break;

        case EnemyState.Chasing:
            if (!isPlayerInChaseBox || PlayerStats.Instance.PlayerIsDead || !hasLOS)
            {
                state = EnemyState.Patrolling;
                rb.linearVelocity = new Vector2(0f, rb.linearVelocity.y);
                animator.SetBool("isWalking", false);
            }
            else if (isPlayerInAttackRange && isAnyAttackReady && hasLOS)
            {
                rb.linearVelocity = new Vector2(0f, rb.linearVelocity.y);
                animator.SetBool("isWalking", false);
                if (ShouldBlock())
                    state = EnemyState.Blocking;
                else
                    state = EnemyState.Attacking;
            }
            else
            {
                ChasePlayer();
            }
            break;

        case EnemyState.Attacking:
            if (isAttacking) return;
            if (!isPlayerInAttackRange || !isAnyAttackReady || PlayerStats.Instance.PlayerIsDead || !hasLOS)
                state = EnemyState.Chasing;
            else
            {
                animator.SetBool("isWalking", false);
                rb.linearVelocity = new Vector2(0f, rb.linearVelocity.y);
                Attack();
            }

            break;

        case EnemyState.Blocking:
            FlipToPlayer();
            if (!isBlocking)
            {
                animator.SetBool("isWalking", false);
                rb.linearVelocity = new Vector2(0f, rb.linearVelocity.y);
                StartBlock();

            }
            break;
    }
}
TilemapParallax.csC# / Unity
// Updates per-group tile parallax by smoothing toward a camera-driven offset,
// then applying a TRS matrix to active tiles while restoring removed tiles.
private void UpdateGroupParallax()
{
    foreach (TileGroup group in tileGroups)
    {
        if (!IsGroupVisibleOnCurrentFloor(group))
            continue;

        // Smooth offset toward camera-based target
        Vector3 targetOffset = ComputeTargetOffset(group);
        float k = 1f - Mathf.Exp(-followLerpSpeed * Time.deltaTime);
        group.currentOffset = Vector3.Lerp(group.currentOffset, targetOffset, k);

        bool offsetChanged =
            (group.lastAppliedOffset - group.currentOffset).sqrMagnitude > OffsetEpsilonSqr;

        // Nothing active and nothing applied -> reset offset
        if (group.activeTiles.Count == 0 && group.appliedTiles.Count == 0)
        {
            group.currentOffset = Vector3.zero;
            group.lastAppliedOffset = new Vector3(float.NaN, float.NaN, float.NaN);
            group.parallaxBlend = 0f;
            continue;
        }

        // Remove tiles no longer active
        if (group.appliedTiles.Count > 0)
        {
            group.removalBuffer.Clear();
            foreach (var t in group.appliedTiles)
            {
                if (!group.activeTiles.Contains(t))
                    group.removalBuffer.Add(t);
            }
            foreach (var t in group.removalBuffer)
            {
                tilemap.SetTransformMatrix(t, group.tileOriginalMatrices[t]);
                group.appliedTiles.Remove(t);
            }
        }

        // Compute parallax matrix once per frame (always includes current offset)
        Matrix4x4 parallaxMatrix =
            Matrix4x4.TRS(group.currentOffset, Quaternion.identity, Vector3.one);

        // Apply parallax to all applied tiles (keep original flips/rotation)
        foreach (var t in group.appliedTiles)
        {
            tilemap.SetTransformMatrix(t, parallaxMatrix * group.tileOriginalMatrices[t]);
        }

        // Apply parallax to newly active tiles
        foreach (var t in group.activeTiles)
        {
            if (!group.appliedTiles.Contains(t))
            {
                tilemap.SetTransformMatrix(t, parallaxMatrix * group.tileOriginalMatrices[t]);
                group.appliedTiles.Add(t);
            }
        }

        group.parallaxBlend = group.isActive ? 1f : 0f;
        group.lastAppliedOffset = group.currentOffset;
    }
}
PlayerStats.Status.csC# / Unity
// Applies status buildup with resistance scaling, blocks buildup if the effect is already active,
// and triggers the status once the threshold is reached (either holding at full or resetting).
public bool AddStatusBuildup(StatusAilment ailment, float rawAmount)
{
    if (PlayerIsDead) return false;
    if(pc.Movement.IsInvincible) return false;

    EnsureBuildupInit();

    // If an effect is currently active, block new buildup for that ailment
    if (_activeEffectLock.Contains(ailment))
        return false;

    // More resistance => slower fill
    float add = rawAmount * StatusBuildupMultiplier(ailment);
    float before = _buildup[ailment];
    float after = Mathf.Min(statusThreshold, before + Mathf.Max(0f, add));

    if (!Mathf.Approximately(after, before))
    {
        _buildup[ailment] = after;
        OnStatusBuildupChanged?.Invoke(ailment, after, statusThreshold);
    }

    if (after >= statusThreshold)
    {
        // Trigger effect
        TriggerStatus(ailment);
        // Notify UI
        OnStatusTriggered?.Invoke(ailment);

        if (HoldAtFullOnTrigger(ailment))
        {
            // Keep bar full and start a controlled decay driven by the effect's duration coroutine
            _buildup[ailment] = statusThreshold;
            OnStatusBuildupChanged?.Invoke(ailment, statusThreshold, statusThreshold);
            // Note: the corresponding coroutine (burn/poison/arcane) will take ownership of decaying the bar.
        }
        else
        {
            // Instant effects reset immediately
            _buildup[ailment] = 0f;
            OnStatusBuildupChanged?.Invoke(ailment, 0f, statusThreshold);
        }
        return true;
    }

    return false;
}

How Elden Ring Redefined Open World Design

Elden Ring succeeds where many open world games fail because it treats exploration as a reward system rather than a content delivery mechanism. The player is never told where to go — they are released into a world that is visually legible. A distant tower, a glowing tree on the horizon, a fog wall at the end of a dungeon: these are not waypoints but invitations.

The most important design lesson for Graveborne is Elden Ring's approach to optional difficulty. Nothing is locked behind a single path. Players who struggle with one boss can explore elsewhere, grow stronger, and return. This is what I am building into the Grave system — interconnected areas that open lateral routes rather than forcing a linear gauntlet.

The best souls-like challenge is not "this enemy is hard" but "this enemy teaches you something." Fromsoft designs enemies as information, not obstacles.

— Personal design note, 2024

Tolkien's Mythology as a World-Building Template

What makes Middle-earth feel ancient and real is not the volume of lore — it is the layering of consequence. Every ruin has a reason. Every name has an etymology. The Elves remember what Men have forgotten. This creates a world that feels like it existed before the story began.

This directly shapes how Velmir is built in Graveborne. The kingdom did not become corrupt overnight — the king's pact was made generations ago. The graveyard Ashar explores are filled with knights who died in a war that has since been rewritten by historians. Item descriptions are written from unreliable perspectives, just as Tolkien's appendices contain contradicting accounts.

The most transferable technique is what I call mythology layering: build the history of your world first, then subtract from it. What the player finds should feel like the surface of something much deeper — because it is.

RDR2 and the Art of Environmental Storytelling

Red Dead Redemption 2 almost never uses cutscenes to deliver its emotional core. The clearest example: the camp at Horseshoe Overlook early in the game is alive with laughter and argument. Return to the same location near the end and it is abandoned. No text. No notification. The world tells the story.

This is the gold standard for environmental storytelling and it directly informs how Graveborne handles its Grave system. Each Grave in Velmir has a lore inscription when you first discover it. After certain boss is defeated, those inscriptions change — subtly, in ways a careful player notices and a casual player misses.

The design principle I extract: the player should be able to construct the story from observation alone. Dialogue and text are supplements, not foundations. A player who reads nothing in Graveborne can still feel the weight of Velmir's tragedy — they just won't know its name.
